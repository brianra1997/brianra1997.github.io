PUSH

User Interface Design Project

Role: Project Timeline:
Interaction/UI Designer 10 Weeks (March 2020 - May 2020)

Premise

For my User Interface Design II class, I sought out to create a mobile application with the intent to teach a new skill to an audience. PUSH is a skateboard tutor, catered to teach people of all levels specific tricks and skills to advance them to the next course. These courses start from basic skills such as learning how to ride a skateboard, to more intermediate topics such as learning how to Ollie (getting the board off the ground without using your hands). The goal of this application is to teach people how to skateboard, while introducing less commonly known techniques to further enhance their experience.

We grouped similar ideas and found the following:

  • Create an experience that is friendly and patient (skateboarding is not an easy skill).
  • Understanding what a beginner already understands to remove extraneous information.
  • Creating an interface that appeals to learning, while also keeping the user engaged.
  • Teaching in a way that allows for information to reach user's long term memory from their working memory.

PUSH final mockup

Creating a Learning Environment

After gathering research, I made narrowed down the features I wanted to integrate into my prototype. These features were chosen specifically to enhance learning, as well as memory retention.

Site-mapping

Once I had an idea of the features I wanted to integrate into my prototype, I started mapping out my application's hierarchy. This was an easy way to begin visualizing how the application could potentially look.

Creating a sitemap allows for easy visualization of an application's hierarchy before development.

I realized that the screens could be simplified to 3 main categories: the home screen, course screen, then the actual tutorial screens. I wanted to emphasize the use of progress bars not only to track users' progression, but also because users tend to learn better when tasks are seen to be unfinished (Zeigarnik effect). Seeing the navigation in this form allowed for an easy transition to the next step of the process: wireframing.

Wire-framing

Once I figured out the general navigation for my application, I started wire-framing. This allowed me to see my application with the interaction hierarchy before adding content. Wire-framing is effective for it forces you to focus on the composition and layout of your screens, as well as navigation without being sidetracked by filling in content/visual design elements (color, pictures, etc.).

PUSH wireframe

Medium-Fidelity Prototype

Once I got the navigation and interaction hierarchy figured out, I began adding content as well as visual design elements. I decided to use minimal colors to bring more focus to the images and their corresponding instructions (tutorial screen). I also made sure to use visual cues (arrows, back button, indicators) to further guide the user through the application.

Medium fidelity prototype: Home, courses, trick-tip screens.
